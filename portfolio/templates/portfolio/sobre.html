Sobre esta Aplicação

1. Arquitetura Django MVT

Esta aplicação segue o padrão MVT (Model – View – Template) do Django. O Model define a estrutura dos dados e comunica com a base de dados via ORM. A View recebe o pedido HTTP, consulta os modelos necessários e decide qual template renderizar. O Template recebe o contexto da view e produz o HTML final enviado ao browser.

Fluxo: o browser envia um pedido → o URL dispatcher chama a view correta → a view consulta o(s) modelo(s) → passa os dados ao template → o template gera o HTML → resposta ao browser.

2. Arquitetura Geral do Projeto

A aplicação corre num container Docker que agrega o servidor Django (Gunicorn) e os ficheiros estáticos (WhiteNoise). O container liga-se a dois serviços externos: a base de dados PostgreSQL na Neon (cloud) e o armazenamento de ficheiros media no Cloudinary. Em desenvolvimento, o container é executado localmente.

Quando as alterações estão prontas, são copiadas para o repositório de deploy e feito push para o GitHub. Esse push dispara automaticamente um workflow de GitHub Actions que constrói a imagem Docker e faz deploy para o servidor de produção da DEISI (Universidade Lusófona), onde a aplicação fica acessível publicamente.
